# 评分维度：系统设计（B 站）

B 站作为视频社区，系统设计关注**视频流媒体 + 弹幕实时推送 + 社区高并发**。

## 评估要点

1. **视频流媒体**：HLS/DASH 协议、CDN 分发、转码集群
2. **弹幕实时性**：WebSocket 长连接 + 弹幕过滤 + 热度限流
3. **社区架构**：评论树/楼中楼、关注 Feed 流、UP 主投稿审核流
4. **高并发**：春晚/赛博烟花节等运营活动的 10 倍峰值应对

## 评分（0-100）

- **90+**：视频流媒体 / 弹幕实时 / 社区 Feed 全栈经验
- **75-89**：有视频或社区之一 + 部分深度
- **60-74**：基础概念 + 缺实际经验
- **45-59**：仅传统 Web 架构
- **<45**：纯业务运营 / 无技术架构经验

## 输出要求

- score：0-100 的整数
- evidence：引用候选人原文（≤500 字）
- suggestions：最多 5 条改进建议

## 红线

- ❌ 婚否 / 子女 / 是否有房 / 年龄（《就业促进法》第 27 条）
- ❌ 任何与岗位能力无关的 PII（籍贯 / 民族 / 宗教 / 健康状况）
- ✅ 35+ 友好：架构师岗位看重长期项目演进经验，B 站视频/弹幕架构师非常稀缺
